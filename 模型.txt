COMMIT 2 — 真 PatchTST：patch-token 流程 + RevIN + mask 友好

你当前 PatchTSTBackbone 是 concat values/mask 后 TransformerEncoder。
这一步把它升级成真 PatchTST（patchify、channel-independent、可长 lookback）。

2.1 替换 models/patchtst.py 为“真 PatchTST backbone”

修改/重建 models/backbones/patchtst.py（不要继续用旧文件名也行，但 registry 里必须能 build）
实现要点（强制）：

RevIN：按 feature/channel 做可逆归一化（train 统计、inference 复用）

Patchify：

输入 [B,L,D] -> 每个 D 独立 patch -> tokens [B, D, N_patches, patch_len]

Patch embedding：Linear(patch_len -> d_model)

Positional embedding：对 patch index

Encoder：标准 Transformer encoder over patch tokens（按 channel-independent 设计）

Pooling：输出 h_seq（建议输出到 [B,L,d_model] 或 [B,N_patches,d_model]，再由 head 统一处理）

mask 融合：

mask patch 化后作为额外通道 或 attention bias（避免“零值=缺失”的混淆）

新增 tests/test_patchtst_patchify.py

L 必须能被 patch_len 整除，否则策略：pad 到最近整除（pad 的位置 mask=1）

断言 patch 数正确、pad mask 正确

✅ Done：PatchTST 不再是 concat-mask 的“伪 PatchTST”

PatchTST patch 设计的参考：HF 文档与官方实现（你写 code 时可以对照）。

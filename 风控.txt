全局设计原则（Codex 必须照做）
Rule-1：内部状态是“一等公民”

任何下单决策必须依赖 AccountState（余额/仓位/保证金/未完成订单/最近成交滑点/资金费率等）。没有 state 或 state 过期 → 进入保护模式（通常是冻结新单或极度缩尺度）。

Rule-2：不变怂：硬底线少拦，软惩罚调尺度

硬底线（Hard Limits）：只拦“会死/失控/状态不可信”。

软惩罚（Soft Penalties）：不拦交易，调整仓位、band、成本、执行风格。

Rule-3：执行延迟不是“等下一根 bar”

延迟只允许发生在秒级（0–60s）用于 TWAP 切片/等待 ack/等待对账，不允许把信号延迟到下一根 5m 才执行。

Rule-4：训练损失“学分布”，策略层做确定性优化

模型学 μ/σ（或分位数）+ 方向概率（可选），策略层用 risk budget / band / cost 进行确定性下单决策。避免端到端可微 Sharpe/PnL 直接当损失（不稳定且容易学偏）。

COMMIT 5 — RiskManager：硬底线 + 软惩罚（Green/Yellow/Red）
5.1 新增 backend/risk_manager.py

入口：

evaluate(account: AccountState, symbol, order_intent, market_snapshot) -> RiskState

硬底线（Hard Limits）

account.health.is_fresh == False → RED（冻结）

margin_ratio < MARGIN_RED → RED（冻结+强制降风险建议）

free_margin < MIN_FREE_MARGIN_USD → RED

positions drift / recon_ok=False → RED

reject_rate_5m > REJECT_RED → RED（执行不可用）

软惩罚（Soft Penalties）

输出连续因子：

pos_scale in (0,1]（仓位缩放）

band_scale >= 1（band 放大，减少噪声交易）

cost_scale >= 1（成本上调，更谨慎）

exec_style_bias（偏向 passive 或 marketable）

触发条件示例（YELLOW，不冻结）：

slippage_p90_bps > SLIP_YELLOW → pos_scale *= 0.7, cost_scale *= 1.3, exec_style_bias='passive'

vol_spike → band_scale *= 1.5（更少噪声单，但大信号仍能穿过）

margin_ratio between YELLOW/RED → pos_scale *= 0.5

COMMIT 6 — PositionSizer：把模型输出（μ/σ/分位数）+ 成本 + 风控 变成目标仓位
6.1 新增 backend/position_sizer.py

核心函数：

compute_target_position(pred, cost_map, account, risk_state, params) -> (target_pos, band, order_intent)

推荐默认（生产友好）：

score_h = (mu_h - cost_h) / (sigma_h + eps)（或用分位数宽度替代 sigma）

score = sum_h w_h * score_h

base_target = clip(k * score, -pos_max, pos_max)

target = base_target * risk_state.pos_scale

band = base_band * risk_state.band_scale（confidence 越低 band 越宽）

trade if |target - current| > band

输出 order_intent：

delta_pos = target - current

转成 qty（按资金、合约乘数、最小下单量）

生成 reduce_only（若是强制降风险）

6.2 新增 tests/test_position_sizer_monotonic.py

cost 增大 → target 应变小

risk_state.pos_scale 降低 → target 应同比降低
